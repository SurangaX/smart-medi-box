Smart Medi Box API Diagnostics

Use these links to verify that the API router is correctly handling modules and bypassing path prefixes like robot_api.

System Status

GET /api/status (Standard path)
GET /robot_api/api/status (With redundant prefix)

SMS Module (New)

To test SMS sending, you should use POST, but you can check the 404 handler for the module via GET:

GET /api/sms/check (Should return SMS action not found)
GET /robot_api/api/sms/check (Should also return SMS action not found, NOT Unknown Module)

Schedule Module

GET /api/schedule/get

Articles Module

GET /api/articles/list

Note: These links assume index.php is in the same directory. If testing on a server with mod_rewrite, you can remove index.php/ from the URLs.
